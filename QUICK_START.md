# 🚀 Быстрый старт: Настройка Telethon для отправки подарков

## 📋 Что нужно сделать (2 простых шага):

---

## Шаг 1: Сгенерировать String Session локально

### 1.1 Остановить Railway бот
Зайти на https://railway.app/ и временно остановить деплой (чтобы не было конфликта)

### 1.2 Запустить генератор сессии
```bash
python generate_session.py
```

### 1.3 Ввести код из Telegram
Telegram отправит код на номер `+998971051000`

Введите код:
```
Please enter the code you received: 12345
```

### 1.4 Скопировать string session
После успешной авторизации в терминале появится:
```
✅ Успешная авторизация!

============================================================
📝 Ваша STRING SESSION (скопируйте в Railway):
============================================================
1BVtsOJABu7G5L... (очень длинная строка) ...
============================================================
```

**СКОПИРУЙТЕ ЭТУ СТРОКУ!**

---

## Шаг 2: Добавить в Railway Environment Variables

### 2.1 Открыть Railway проект
1. Зайти на https://railway.app/
2. Открыть проект `StarPayUzAuto`
3. Перейти в **Variables** (или Settings → Variables)

### 2.2 Добавить эти переменные:

```
API_ID=30654977
API_HASH=921be05f47930bd6e60860faa4c6b0d5
TELETHON_SESSION_STRING=<вставить_скопированную_строку>
```

**Где взять значения:**
- `API_ID` и `API_HASH` — уже в вашем .env файле
- `TELETHON_SESSION_STRING` — скопировали на Шаге 1.4

### 2.3 Сохранить и перезапустить
Railway автоматически перезапустит бот с новыми переменными.

## ❓ Troubleshooting

### "Telethon client not connected"
→ Проверьте что `TELETHON_SESSION_STRING` правильно добавлена в Railway Variables

### "FloodWait"
→ Слишком много запросов. Подождите указанное время (обычно 30-60 секунд)

### "Username not found"
→ Проверьте что username правильно указан (без @)

### Бот не запускается
→ Проверьте логи Railway на наличие ошибок
